# 功能清单

> 最后更新：2026-04-08

## 核心功能模块

### 1. 专题网站编辑器

所有专题统一为网站形式，通过 AI Agent 辅助对话创建。

#### 1.1 专题基础管理

| 功能点 | 输入/触发 | 输出/结果 | 交互流程 | 错误处理 |
|--------|----------|----------|---------|---------|
| 创建专题 | 用户ID、标题、描述 | 专题ID、自动进入编辑器 | 用户填写标题和描述→系统创建记录→跳转到编辑器 | 缺少必填字段提示；权限验证 |
| 编辑专题 | 专题ID | 进入编辑器 | 用户选择专题→打开编辑器界面 | 专题不存在或无权限提示 |
| 发布专题 | 专题ID | 发布成功 | 用户点击发布→系统更新状态→**所有人可见** | 未发布专题仅editor可见 |
| 关闭专题 | 专题ID | 关闭成功 | 用户点击关闭→更新状态为已关闭 | 关闭后标记为已关闭 |
| 查看专题列表 | 无需登录 | 专题列表 | **所有人（包括访客）查看所有已发布的专题** | 无专题时显示空状态 |
| 查看专题详情 | 专题ID | 网站预览 | 直接预览专题网站 | **已发布的专题对所有人公开** |
| 删除专题 | 专题ID | 删除成功 | 用户点击删除→确认→清理文件和数据库记录 | 权限验证 |

#### 1.2 网站编辑器（VSCode风格三栏布局）

| 功能点 | 输入/触发 | 输出/结果 | 交互流程 | 错误处理 |
|--------|----------|----------|---------|---------|
| Agent对话生成网站 | 用户在对话区描述需求 | 生成的网站代码和预览 | 教师描述需求→Agent询问偏好→生成代码→写入WebContainer→预览更新 | LLM服务不可用提示 |
| 文件上传作为参考 | 用户上传Markdown/Word/PDF等文件 | 文件内容被Agent解析 | 教师上传文件→Agent解析内容→作为网站生成的参考 | 文件格式验证 |
| 查看和编辑文件树 | 左侧面板 | 文件列表和结构 | 教师查看Agent生成的文件→双击打开编辑器→直接修改代码 | 文件操作权限验证 |
| 实时预览网站 | WebContainer自动热更新 | iframe中显示网站 | 代码修改后WebContainer自动热重载→预览区实时更新 | WebContainer初始化失败提示 |
| 保存文件快照 | 点击保存按钮 | 文件和对话历史保存到后端 | 系统同步所有文件到数据库→保存对话历史 | 网络错误提示 |
| 发布到网站 | 点击发布按钮 | 网站对外公开可访问 | 打包文件→发布到外部存储→生成访问链接 | 发布失败提示 |
| 生成分享链接 | 点击分享按钮 | 临时预览链接 | 生成可分享的URL | 链接过期提示 |
| 响应式预览切换 | 选择设备类型 | 不同设备尺寸的预览 | 切换桌面/平板/手机模式→预览区调整iframe宽度 | 无 |

**编辑器布局：**
- **左侧面板（18%默认）**：文件系统树，支持展开/折叠、文件操作（新建、重命名、删除）
- **中间面板（28%默认）**：Agent对话区，支持文本输入、文件上传、代码块渲染
- **右侧面板（自适应）**：浏览器预览（iframe），WebContainer在后台运行
- 所有面板支持拖拽调整宽度，参考VSCode风格

**WebContainer支持：**
- StackBlitz WebContainers API
- 完整Node.js环境，支持npm install和构建工具
- 支持React、Vue、Svelte等常见前端框架
- 文件修改通过fs.writeFile更新，自动热重载

### 2. AI 学习助手

> **设计说明：**
> - AI 助手集成在专题空间侧边栏，用户可随时唤出
> - 助手具有查询当前专题内容的能力，提供上下文相关的学习支持
> - **仅注册用户可用**：学生和教师可使用 AI 助手，访客需注册登录
> - **只读型 Agent**：不修改业务数据，专注于理解与查询
> - 详细设计见 [AI Agent 系统](./agents.md)

| 功能点 | 输入/触发 | 输出/结果 | 交互流程 | 错误处理 |
|--------|----------|----------|---------|---------|
| 唤出 AI 助手 | 点击侧边栏按钮 | AI 对话窗口打开 | 用户点击侧边栏"AI助手"按钮→展开对话窗口→显示欢迎消息 | 网络错误提示 |
| AI 对话问答 | 用户问题文本、topic_id | AI 回复、相关内容引用 | 用户输入问题→AI 分析意图→调用工具查询专题内容→生成带上下文的回复→引用相关页面/段落 | AI 服务不可用提示 |

**适用场景：**
- 学生：概念解释、内容定位、进度查询
- 教师：专题回顾、学习跟踪、课堂准备

> **详细实现：** 学习助手 Agent 的工具定义（OpenAI Function Calling 格式）、交互流程详见 [AI Agent 系统](./agents.md#1-学习助手-agent)

### 3. AI 专题搭建助手

> **设计说明：**
> - 入口在网站编辑器的中间面板对话区
> - **主动协作模式**：Agent先询问偏好（风格、布局、颜色等），再生成网站代码
> - 通过 WebContainer FS API 直接写入文件，即时预览验证
> - 详细设计见 [AI Agent 系统](./agents.md)

| 功能点 | 输入/触发 | 输出/结果 | 交互流程 | 错误处理 |
|--------|----------|----------|---------|---------|
| 唤出搭建助手 | 打开编辑器，自动加载 | Agent对话窗口就绪 | 用户描述需求→Agent询问偏好→生成网站代码→写入WebContainer | 权限验证；网络错误提示 |
| 生成网站代码 | 主题描述、偏好选择 | 生成的完整网站 | 用户描述需求→Agent确认偏好→生成代码→写入文件→预览更新 | LLM服务不可用提示 |
| 修改现有网站 | 对话区提出修改要求 | 更新后的网站 | 用户描述修改→Agent理解上下文→修改文件→预览更新 | 上下文理解失败提示 |
| 多轮对话迭代 | 连续对话 | 逐步完善的网站 | 用户预览效果→提出修改意见→Agent迭代修改→直到满意 | 对话历史不丢失 |

**适用场景：**
- 新建专题：通过对话描述需求，Agent生成完整网站
- 编辑专题：提出修改要求，Agent调整代码
- 发布前检查：预览效果，确认无误后发布

> **角色说明：** 此前的教师/学生角色已统一为 `user`，专题级别的编辑权限通过 `editors` 字段控制。

> **详细实现：** 专题搭建 Agent 的 Prompt 结构、LLM API 接口、文件操作流程详见 [AI Agent 系统](./agents.md#2-专题搭建-agent)

## 设计理念

### 简化原则

1. **统一专题模型**：取消知识型/网站型双版本，所有专题都是网站形式
2. **AI 驱动创建**：教师通过对话创建网站，无需技术背景
3. **公开访问**：所有已发布专题对访客开放，降低参与门槛
4. **AI 辅助**：学习助手帮助理解，搭建助手帮助创建

### 用户体验

1. **访客友好**：无需注册即可浏览专题内容
2. **即时反馈**：WebContainer 实时预览，代码修改立即可见
3. **VSCode 风格**：熟悉的编辑器界面，降低学习成本
4. **AI 赋能**：通过 AI 降低内容创建门槛
