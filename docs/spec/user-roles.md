# 用户角色与权限

> 最后更新：2026-04-08

## 角色定义

### 访客（未登录用户）

**描述：** 访问平台的任何人，无需注册登录

**主要功能：**
- 查看所有已发布的专题学习空间
- 查看专题详情和页面内容
- 注册账号

### 用户（已注册用户）

**描述：** 已注册的专题学习参与者（原学生/教师角色已统一为 `user`）

**主要功能：**
- 所有访客权限
- 使用 AI 学习助手帮助理解专题内容
- 使用 AI 专题搭建助手创建和编辑专题（具有 editors 权限的专题）
- 需要登录认证

> **历史：** 此前系统区分了 `teacher` 和 `student` 角色。为简化权限模型，两者已统一为 `user` 角色。专题级别的协作权限通过 `editors` 字段（JSON 数组）实现。

## 权限矩阵

### 专题管理权限

| 操作 | 访客 | 用户（非editor） | 用户（editor） | admin |
|------|:----:|:----------------:|:--------------:|:-----:|
| 查看已发布专题列表 | ✓ | ✓ | ✓ | ✓ |
| 查看专题详情 | ✓ | ✓ | ✓ | ✓ |
| 创建专题 | ✗ | ✓ | ✓ | ✓ |
| 编辑专题 | ✗ | ✗ | ✓ | ✓ |
| 发布/关闭专题 | ✗ | ✗ | ✓ | ✓ |

### AI Agent 权限

| Agent 类型 | 访客 | 用户（非editor） | 用户（editor） | admin |
|------|:----:|:----------------:|:--------------:|:-----:|
| 学习助手 Agent（learning） | ✗ | ✓ 所有已发布的 | ✓ | ✓ |
| 专题搭建 Agent（building） | ✗ | ✗ | ✓ 有编辑权限的专题 | ✓ |

## 权限说明

### 访客权限（完全公开）

- 可以查看所有已发布的专题列表
- 可以查看任何已发布专题的详情和页面内容
- **无需登录即可浏览专题内容**

### 用户权限

- 所有访客权限
- 可以使用 AI 学习助手理解专题内容、查询进度
- 可以创建专题
- 可以通过 `editors` 字段获得其他用户创建的专题的编辑权限
- 需要登录认证

### Admin 权限

- 所有用户权限
- 管理所有专题（不受 editors 限制）

## 公开访问原则

### 核心原则

1. **完全公开浏览**
   - 所有已发布的专题对所有人公开可见
   - 访客无需登录即可自由浏览
   - 降低参与门槛，促进知识共享

2. **分级操作权限**
   - 浏览专题：公开访问（访客、用户、admin）
   - AI 学习助手：需要登录（用户、admin）
   - 专题管理：需要登录（创建者、editors、admin）
   - AI 专题搭建：需要登录且有编辑权限

3. **基于 editors 的协作**
   - 专题创建者自动成为 editor
   - 可通过编辑专题的 `editors` 字段添加协作者
   - Editor 可以看到草稿状态专题并参与编辑

### 设计理念

- **开放优先**：默认公开，促进知识传播
- **简单直接**：访客即可浏览，无需复杂流程
- **AI 赋能**：学习助手帮助理解，搭建助手帮助创建
- **按需认证**：只在需要 AI 或管理操作时要求登录
